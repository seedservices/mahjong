# AGENTS.md — Project Instructions

## Purpose
This repository uses AGENTS.md to keep working instructions for coding agents.

## Canonical Documentation
When asked for product behavior, requirements, or rules, use the following as the source of truth:
- `docs/functional-spec.md` for user-facing behavior and requirements.
- `docs/technical-spec.md` for architecture, data structures, and implementation details.
- `docs/user-guide.md` for end-user procedures and UI behavior.

If these documents conflict with code, report the discrepancy and ask whether to update docs or code.

## Language and Formatting
- Default to concise, technical English.
- Use Markdown for new documentation.
- Avoid non-ASCII unless the file already uses it.

## Scope Defaults
If the user does not specify scope, assume the full single-hand Hong Kong Mahjong app.
